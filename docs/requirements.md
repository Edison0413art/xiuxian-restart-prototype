# 《修仙重开模拟器》需求文档

版本：v0.2 原型版  
更新时间：2026-05-26  
当前原型地址：http://localhost:4173/

## 1. 项目定位

《修仙重开模拟器》是一款手机网页文字模拟小游戏，核心体验参考《人生重开模拟器》，主题改为修仙、网络热梗、小说梗和离谱重开。

玩家从 0 岁出生开始，选择天赋、分配属性、逐年经历随机事件，并在每一年做出选择。游戏重点不是复杂战斗，而是高随机、高梗密度、高死亡率和多结局带来的“再来一局”体验。

核心卖点：

- 修仙版人生重开
- 0 岁开局，随时可能暴毙
- 网络热梗、修仙爽文、重生文、霸总文、番茄流乱炖
- 左韩天尊作为贯穿全局的超级 Boss
- 本地事件表驱动主流程
- DeepSeek 只做结局页额外文案，不参与主逻辑

## 2. 当前版本范围

首版原型目标：

- 1000 条高质量样例事件
- 100 个结局
- 不做长期存档
- 不做复杂战斗系统
- 不做公开上线级版权处理
- 不让 AI 决定数值或事件结果

后期目标：

- 10000 条事件
- 500 个结局
- 更完整的剧情线、结局图鉴、分享图、移动端动效和数据配置后台

## 3. 核心流程

1. 首页  
   玩家进入游戏，输入道号。留空则随机生成。

2. 天赋选择  
   随机抽取 10 个天赋，玩家选择 3 个。

3. 属性分配  
   玩家获得 20 点，分配到 6 个基础属性。

4. 出生判定  
   从 0 岁开始，极端加点可能直接触发出生暴毙。

5. 年度事件  
   每个时间节点随机出现 2-4 个选项。玩家点击一个选项后，显示事件结果和属性变化。

6. 修仙推进  
   根据修为和属性判定突破境界。境界越高，寿元越高，时间跨度越大。

7. 死亡或终局  
   玩家可能因寿元耗尽、事件暴毙、突破失败、左韩路过、飞升等进入结局。

8. 结局页  
   显示本局结局、战报、标签、图鉴、人生履历和“请天道加戏”按钮。

## 4. 基础属性

开局 20 点分配到以下属性：

| 属性 | 作用 |
|---|---|
| 灵根 | 影响修炼速度、突破概率、拜入仙门概率 |
| 悟性 | 影响功法学习、顿悟、特殊事件收益 |
| 体魄 | 影响生存率、抗暴毙能力、突破失败承受力 |
| 家底 | 影响初始灵石、资源事件、富二代路线 |
| 气运 | 影响奇遇、掉宝、死亡概率、左韩事件风险 |
| 颜值 | 影响退婚流、霸总串台、天妒红颜事件 |

动态数值：

| 数值 | 说明 |
|---|---|
| 年龄 | 当前年龄 |
| 境界 | 当前修仙境界 |
| 修为 | 当前境界内进度 |
| 寿元 | 当前境界寿命上限 |
| 灵石 | 资源货币 |
| 名声 | 影响主角乱入和社会事件 |
| 左韩注视 | 越高越容易触发左韩相关事件 |

## 5. 境界与寿元

境界采用正统修仙等级：

| 境界 | 寿元上限 |
|---|---:|
| 凡人 | 80 |
| 炼气 | 120 |
| 筑基 | 200 |
| 金丹 | 500 |
| 元婴 | 1000 |
| 化神 | 3000 |
| 炼虚 | 8000 |
| 合体 | 20000 |
| 大乘 | 50000 |
| 渡劫 | 90000 |

时间推进规则：

- 凡人期：一年一跳
- 炼气/筑基：1-3 年一跳
- 金丹：5-10 年一跳
- 元婴及以上：几十年到数千年一跳
- 渡劫后进入飞升或终局判定

## 6. 选项设计

每年随机出现 2-4 个选项。

当前产品规则：

- 选项卡只显示标题、分类和事件描述
- 不在选项卡上显示“倾向 / 风险 / 可能影响”
- 玩家点击后才显示具体结果和属性变化
- 选项之间避免同一轮出现重复标题

原因：

- 保持文字模拟的未知感
- 避免界面信息过载
- 让玩家通过剧情文本判断风险，而不是看数值标签做最优解

点击后显示：

- 事件标题
- 事件结果文案
- 属性变化，例如 `修为 +7`、`气运 -1`
- 突破、死亡、剧情线推进等特殊结果

## 7. 左韩天尊机制

左韩天尊是本作超级 Boss，但不再高频本体出现。当前版本拆成三层：

| 层级 | 说明 | 频率 | 危险度 |
|---|---|---|---|
| 左韩传闻 | 听见传闻、看到痕迹、小地图闪烁 | 常见 | 低 |
| 左韩余波 | 远处事件波及玩家 | 中等 | 中 |
| 左韩本体路过 | 左韩天尊亲自刷新到附近 | 稀有 | 极高 |

设计目标：

- 保留左韩压迫感
- 避免 Boss 过度出现导致掉价
- 让“左韩本体路过”成为真正高冲击事件
- 反讽“为什么最终 Boss 不在主角弱小时毁灭主角”

左韩相关状态：

- 左韩注视值
- 左韩传闻次数
- 左韩余波次数
- 左韩本体路过次数
- 结局页显示“左韩关系”

## 8. 剧情线系统

当前版本包含 6 条剧情线。事件可以推进剧情线进度，最多 5 阶段。

| 剧情线 | 阶段方向 |
|---|---|
| 退婚三年之约线 | 被看不起 -> 三年之约 -> 暗中发育 -> 反手打脸 |
| 戒指老爷爷夺舍线 | 戒指发热 -> 老爷爷开机 -> 开始教学 -> 疑似夺舍 -> 反向收编 |
| 左韩注视线 | 听见传闻 -> 遭遇余波 -> 被注视 -> 本体路过 -> 终局对峙 |
| 颜值天妒线 | 建模过高 -> 路人围观 -> 天道警告 -> 雷劫定位 |
| 凡人苟道线 | 先活下来 -> 能跑就跑 -> 绕开主线 -> 苟出名堂 |
| 霸总串台线 | 眼神凉薄 -> 办公室飞升 -> 短剧爆款 -> 合同成道 |

游玩中如果已有剧情线进度，会显示“剧情线”面板。

## 9. 事件池规划

首版 1000 条事件：

| 分类 | 数量 |
|---|---:|
| 出生/家庭/童年/早死 | 90 |
| 少年期/测灵根/拜师/入宗 | 90 |
| 凡人江湖/村庄/市井/小地图 | 70 |
| 炼气期日常与小奇遇 | 110 |
| 筑基期秘境/宗门/散修线 | 110 |
| 金丹期装逼/夺宝/翻车 | 90 |
| 元婴及以上大事件 | 90 |
| 左韩天尊事件链 | 80 |
| 热门小说主角乱入 | 80 |
| 霸总/重生/番茄/都市串台 | 60 |
| 法宝/戒指/老爷爷/系统流 | 70 |
| 特殊死亡/隐藏暴毙 | 40 |
| 飞升/终局/特殊路线触发 | 20 |
| 合计 | 1000 |

注意：

- 热门小说主角不应在所有事件里高频出现
- 主角名字只在“热门小说主角乱入”类事件中高概率出现
- 普通事件主要使用网络梗、修仙梗和小剧情
- 梗密度要有节奏，不能每句话都硬塞多个梗

## 11. 本局战报

结局页做成更有传播感的战报结构：

- 享年
- 最高境界
- 死因/终局
- 左韩关系
- 本局关键词
- 一句话锐评
- 复制战报按钮

示例：

```text
《修仙重开模拟器》本局战报
道号：林无敌
享年：180 岁
最高境界：金丹
死因/终局：金丹庆典遭遇天榜留名
左韩关系：吃到余波 1 次
关键词：左韩相关、戒指老爷爷、金丹、退婚三年之约线
锐评：主线味最重的是：戒指老爷爷夺舍线。
```

## 12. 本局图鉴

不做长期存档，但做当前浏览器会话内的本局图鉴。

图鉴包含：

- 见过结局
- 见过死法
- 走过剧情线

规则：

- 刷新页面后可以丢失
- 不需要账号
- 不需要 localStorage
- 用于增强“再来一局”的短期收集感

## 13. DeepSeek API

DeepSeek 只用于结局页的可选按钮：**请天道加戏**。

用途：

- 根据本局日志生成天道点评
- 根据标签生成结局小作文
- 增加额外搞笑文案

限制：

- 不生成主事件
- 不参与数值结算
- 不决定死亡或结局
- 不影响平衡
- API key 必须放后端，不放前端

没有 API key 时，使用本地兜底文案。

## 15. 技术方案

当前原型：

- 前端：原生 HTML/CSS/JS
- 服务：Node.js HTTP server
- 数据：当前在 `main.js` 中生成
- AI：`server.js` 提供 DeepSeek 代理接口
- 样式：移动端优先，像素复古风

当前文件：

- `index.html`：页面入口
- `styles.css`：视觉样式
- `main.js`：核心游戏逻辑
- `server.js`：本地服务和 DeepSeek 代理
- `docs/requirements.md`：需求文档
- `docs/github-learning-notes.md`：参考项目学习笔记
- `docs/meme-source-notes.md`：热梗素材笔记

后续建议拆分：

- `data/talents.json`
- `data/events.json`
- `data/endings.json`
- `data/routes.json`
- `data/realms.json`
- `src/game-engine.js`
- `src/ui.js`

## 17. 当前产品原则

- 玩家可以暴毙，但要觉得死得有梗
- 左韩可以强，但不能廉价
- 梗可以多，但不能每句都硬塞
- 选择可以未知，但结果必须清楚
- DeepSeek 可以加戏，但不能接管游戏
- 第一版先好玩，再考虑大规模内容生产
